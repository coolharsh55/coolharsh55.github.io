    SHACL tests to detect missing metadata, mostly arising from typos in relations (e.g. parent concept) sometimes seem not to work - any implementation going ahead could have some 'unit test' to check for regressions.

    OWL serialisation can possibly be generated from RDFS+SKOS e.g. using SPARQL CONSTRUCT queries. An important question is whether there will be ANY instances in OWL e.g. for statuses which are a fixed set, but which in RDFS+SKOS can be extended using the SKOS hierarchy. Another important question is whether there are any OWL axioms to import - I currently am not aware of any, and haven't received any from implementing projects e.g. TRAPEZE. But I think this can be added ex-post the generation e.g. using rdflib.
