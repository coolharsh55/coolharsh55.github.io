Repo

The current repo is structured as follows:

  • dpv - the main dpv spec
  • folders for specific extensions: dpv-dga, dpv-gdpr, dpv-legal, dpv-nace, dpv-owl, dpv-pd, dpv-skos, dpv-tech, rights, risk
  • folders for examples, guides and use-cases
  • primer
  • folder to store meetings notes and generation code
  • releases
  • folder to store common diagrams
  • folder with documentation-generator code
  • w3id_config - containing the W3ID config for providing permanent URLs for DPV documents and resources
  • Files: README.md, contributing.md - for contribution guidelines, contributors.json - to list all contributors, LICENSE.md, w3c.json - for w3c operational metadata

In this, each of the dpv and extensions folder has a 'sub-structure' within them as follows:

  • files for that specification
  • modules folder containing individual files for each 'module' or 'part' of the vocabulary e.g. legal bases in DPV

The DPV-SKOS and DPV-OWL folders repeat inside them all the DPV and extension folders in their respective serialisations.
